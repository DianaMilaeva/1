1.	Класс и объект, разница:
•	Класс — это шаблон (чертёж), описывающий структуру и поведение.
•	Объект — это конкретный экземпляр класса, созданный в памяти.
•	Разница: Класс — это тип (понятие), объект — реальная сущность, созданная по этому понятию.
2.	Конструктор класса:
•	Это специальный метод, который автоматически вызывается при создании объекта.
•	Нужен для начальной инициализации полей и выполнения стартовых действий.
3.	Свойства vs поля, ключевые слова:
•	Поле — это просто переменная внутри класса.
•	Свойство — это механизм, управляющий доступом к полю через специальные методы.
•	Ключевые слова: get (для чтения), set (для записи).
4.	Первичный конструктор (C# 12):
•	Позволяет объявить параметры конструктора прямо в скобках после имени класса.
•	Удобство: Уменьшает шаблонный код, параметры доступны во всём классе (не нужно вручную создавать поля).
5.	Инкапсуляция:
•	Это скрытие внутреннего устройства класса и защита данных от некорректного внешнего использования.
•	Нужна для повышения надёжности, модульности и упрощения изменений.
6.	Модификаторы доступа:
•	public — доступен откуда угодно.
•	private — доступен только внутри своего класса.
•	protected — доступен внутри класса и в классах-наследниках.
o	internal — доступен только в той же сборке (проекте).
7.	Наследование, символ:
•	Это механизм создания нового класса на основе существующего (базового). Дочерний класс перенимает функциональность родителя.
•	Символ: двоеточие :.
8.	Вызов конструктора базового класса:
•	Через ключевое слово base(...). Делается это в конструкторе производного класса.
•	Нужно: Чтобы гарантировать правильную инициализацию унаследованной части объекта.
9.	Полиморфизм, ключевые слова:
•	Способность одного и того же метода работать по-разному в разных классах.
•	Ключевые слова: virtual (в базовом), override (в производном).
10.	Разница virtual, override, new:
•	virtual — метод можно переопределить (не обязательно).
•	override — переопределяет виртуальный метод (новый код, полиморфизм).
•	new — скрывает метод базового класса (ломает полиморфизм, метод выбирается по типу переменной).
11. Абстрактный класс:
•	Это неполный класс, который может содержать абстрактные методы (без тела). Объект создать нельзя — можно только унаследовать и реализовать.
12. Интерфейс и отличие от абстрактного класса:
•	Интерфейс — контракт, описывающий что умеет объект (только сигнатуры, нет полей и реализации, кроме статических методов/дефолтных в новых версиях C#).
•	Отличия: интерфейс не хранит состояние (нет полей), множественное наследование (класс — только от одного абстрактного класса), все члены интерфейса по умолчанию public.
13. Множественное наследование в C#:
•	От классов: нет, класс может наследоваться только от одного класса.
•	От интерфейсов: да, класс может реализовывать сколько угодно интерфейсов.
14. Структура (struct) vs класс:
•	Структура — значимый тип данных.
•	Принципиальное отличие: структура хранится в стеке или внутри другого объекта (значимый тип), а класс — в куче (ссылочный тип). Структуры не поддерживают наследование (кроме неявного от ValueType).
15. Ссылочные vs значимые типы:
•	Ссылочные: переменная хранит адрес объекта (куча). Пример: class, string, object, array.
•	Значимые: переменная хранит само значение (стек). Пример: struct, int, float, bool, enum.
•	Отличие: присваивание копирует значение (значимые) или ссылку (ссылочные).
16. Когда что использовать:
•	Структура: маленькие, неизменяемые типы данных (точка, цвет, комплексное число) — меньше накладных расходов.
•	Класс: когда нужен полиморфизм, наследование, большие объёмы данных или изменяемое состояние.
17. Иерархия классов и корневой класс:
•	Иерархия — древовидная структура "предок-потомок".
•	Корень всей иерархии в C#: класс System.Object (alias object).
18. Параметры по умолчанию в конструкторе:
•	Это значения аргументов, используемые, если caller их не указал.
•	Объявляются: public MyClass(int x = 10, string name = "default")
19. Несколько конструкторов:
•	Да, класс может иметь несколько конструкторов (перегрузка конструкторов).
•	Различаются по количеству, типам и порядку параметров.
20. Перегрузка методов (overloading):
•	Несколько методов с одинаковым именем, но разной сигнатурой в одном классе.
•	Компилятор различает по количеству параметров, типам параметров и их порядку (но не по возвращаемому типу).
